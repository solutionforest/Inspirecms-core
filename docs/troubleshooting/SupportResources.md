## Creating Effective Support Requests

When seeking help, please provide:

### For Bug Reports

1. **Clear Description**: What happened vs. what you expected
2. **Version Information**:
   ```bash
   php artisan inspirecms:about
   ```
3. **Steps to Reproduce**: Detailed steps to recreate the issue
4. **Environment Details**:
   - PHP version
   - Database system and version
   - Server OS
5. **Relevant Log Output**: From `storage/logs/`
6. **Screenshots**: If applicable

### For Feature Requests

1. **Clear Description**: What functionality you need
2. **Use Case**: How you would use this feature
3. **Current Workarounds**: How you're handling this currently
4. **Similar Features**: Examples from other systems (if applicable)

### For General Questions

1. **What You're Trying to Accomplish**: Your end goal
2. **What You've Tried**: Methods attempted so far
3. **Specific Documentation Referenced**: Links to docs you've consulted
4. **Code Samples**: Minimal examples demonstrating your issue

## Contributing to InspireCMS

We welcome community contributions:

### Reporting Bugs

1. Check existing issues to avoid duplicates
2. Use the bug report template
3. Include detailed reproduction steps

### Submitting Pull Requests

1. Create a fork of the repository
2. Create a feature branch
3. Follow the [contribution guidelines](../Contributing.md)
4. Submit a pull request with a clear description

### Improving Documentation

Help improve our documentation by:

1. Submitting issues for unclear or missing documentation
2. Creating pull requests with documentation improvements
3. Sharing examples and tutorials with the community
